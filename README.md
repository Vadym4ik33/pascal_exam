# 🎫 Шпаргалка к экзамену по Pascal (По билетам)

---

## 🧭 ЗАДАНИЯ 1-ГО ТИПА: ТЕОРИЯ НА ПАЛЬЦАХ

### 🔹 Билет 3. Линейный поиск vs Бинарный
* **Линейный:** Перебор всех элементов подряд. Работает на **любом** массиве. Медленный.
* **Бинарный:** Деление отсортированного массива пополам. Быстрый ($O(\log N)$).
* *Ловушка препода:* Работает **строго на отсортированном** массиве! Если элементы перемешаны — бинарный поиск использовать нельзя, сначала нужна сортировка.

### 🔹 Билет 4. Переменная и Жесткая типизация
* **Переменная** — подписанная коробка в памяти для хранения данных.
* Основные типы: `integer` (целые числа), `real` (дробные), `string` (текст), `boolean` (True/False).
* *Ловушка препода:* Паскаль не разрешит записать дробь (результат операции `/`) или строку в `integer`. Типы должны совпадать железно.

### 🔹 Билет 7. Разница между циклами
* `for i := A to B do` — знаем точное число шагов (счетчик увеличивается сам).
* `while <условие> do` — цикл с пред-условием. Проверяет истинность *ДО* входа в цикл. Если изначально False — не выполнится **ни разу**.
* `repeat ... until <условие>` — цикл с постусловием. Выполняется *ПОСЛЕ* шага. Гарантированно сработает **минимум 1 раз**. Выходит из цикла, как только условие стало `True`.

### 🔹 Билет 8. Массивы в памяти
* Набор однотипных элементов, расположенных в памяти **одним непрерывным куском**.
* *Ловушка препода:* Обращение к несуществующему индексу (например, попытка записать в `a[6]` при объявленном `array[1..5]`) вызовет критическую ошибку выполнения (**выход за границы массива**).

### 🔹 Билет 9. Передача параметров в подпрограммы
* **По значению (`procedure P(x: integer)`)**: Передается *копия* переменной. Исходная переменная в основном коде защищена и не изменится.
* **По ссылке (`procedure P(var x: integer)`)**: Передается *оригинал* (благодаря ключевому слову `var`). Любые изменения внутри процедуры меняют переменную во всей программе.

### 🔹 Билет 11-12. Рекурсия и Стек
* **Рекурсия** — это когда функция вызывает саму себя внутри собственного тела (принцип матрёшки).
* **Стек** — специальная область памяти («стопка тарелок»), куда компьютер складывает текущие незавершенные шаги рекурсии.
* *Ловушка препода:* Если забыть базовый случай (условие выхода из рекурсии), стек переполнится и программа вылетит с ошибкой **Stack Overflow**.

### 🔹 Билет 14-16. Алгоритмы сортировки
* **Пузырёк (Bubble):** Попарное сравнение соседей. Самые «тяжелые» элементы постепенно всплывают в самый конец массива.
* **Вставками (Insertion):** Элемент из неотсортированной части «вставляется» на свое правильное место в уже упорядоченную левую часть.
* **Выбором (Selection):** Находится минимальный элемент в неотсортированной части и меняется местами с текущим первым элементом.

### 🔹 Билет 18. Строки (`string`)
* Строка в Паскале — это массив символов. Нумерация букв всегда начинается **с 1**. `s[1]` — самый первый символ.
* Длина строки вычисляется функцией `Length(s)`. Сложение строк и чистых чисел напрямую запрещено (`'Результат: ' + IntToStr(x)`).

### 🔹 Билет 31. Оператор `case` (Переключатель)
* Заменяет громоздкие многоуровневые конструкции `if-then-else`. Работает по принципу кнопочного пульта.
* *Ловушка препода:* В `case` можно использовать только дискретные/перечисляемые типы (`integer`, `char`, `boolean`). Вещественные числа **`real` использовать в `case` строго запрещено!**

## 🧮 ЗАДАНИЯ 2-ГО И 3-ГО ТИПА: ФОРМУЛЫ И ШАБЛОНЫ

### 📥 Ввод-вывод данных
* `read(x)` / `write(x)` — ввод/вывод данных на текущей строке.
* `readln(x)` / `writeln(x)` — ввод/вывод с автоматическим **переносом курсора на новую строку**.

### 📐 Встроенная математика
* `Sqr(x)` — возведение числа в квадрат ($x^2$).
* `Sqrt(x)` — квадратный корень ($\sqrt{x}$). **Результат вычисления всегда типа `real`!**
* `Abs(x)` — модуль числа ($|x|$).
* `14 div 4` = `3` (целая часть от деления).
* `14 mod 4` = `2` (остаток от деления, используется для проверки четности: `n mod 2 = 0`).

### 🚨 Шаблоны «копилок» (Правила инициализации)
1.  **Сумма / Количество:** Перед циклом переменные-счетчики **обязательно** обнуляются: `sum := 0; count := 0;`.
2.  **Поиск Max/Min:** Инициализируется строго **первым элементом массива**: `max := a[1];`, а цикл проверки запускается со второго элемента: `for i := 2 to N do`.
3.  **Факториал:** Инициализируется **единицей**: `f := 1;` (иначе при умножении на ноль всё занулится), далее в цикле: `f := f * i;`.
4.  **Проверка на простоту:** По умолчанию выставляется флаг `isPrime := true;`. Если в цикле находится делитель без остатка (`n mod i = 0`), флаг сбрасывается: `isPrime := false; break;`.
